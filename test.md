# Pion DX Task: Enhance the Developer CLI Tool (TypeScript)

You've inherited this CLI from an engineer who has left the team. A new engineer is joining next week and will need to use it — and so will an AI assistant on your team.

Your job: make it production-ready and hand it off properly.

There's no single right answer for this task. We're looking for evidence that you're interested in keeping up with how AI is changing developer workflows — and that you can apply that thinking to tooling.

We're not looking for a fully complete solution, but rather a demonstration of your ability to think through the problem and apply best practices. What matters is that you can explain your thinking, so don't spend too much time on any one task. The direction is there to help you share as much of your understanding as possible, not to constrain you.

---

## Suggested tasks

1. **Fix the bug** — the CLI doesn't work in all cases.
2. **Add flags** — `--dry-run`, `--force`, and `--help`.
3. **Refactor** — structure the code so others can extend it.
4. **Improve logging** — errors and success messages should be useful.
5. **Update the documentation** — README should be enough to get someone started.
6. **AI context** — update `AGENTS.md` so an AI assistant can use this tool effectively.

---

## Bonus

Pick one or more if you think this will add value to the project and explain why:

- Add tests.
- Leave a note on how you'd scale this to support multiple commands.
- Explain how you'd integrate this solution into developer's local environment at scale.
